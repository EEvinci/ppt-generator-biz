# ppt-generator-biz

商业风格 16:9 HTML 演示文稿生成器。

> ⚠️ **重要更新（2026-04-09）：本 Skill 已重构生成方式，详见下方「标准工作流」。旧版 `gen.py` + `template.html` 方案已废弃，请勿使用。**

---

## 标准工作流

```
1. 规划 SLIDES 内容（~20页）
   ↓
2. 编写 build.py（或用 dist/gen.py 直接生成）
   ↓
3. 运行：python3 build.py
   ↓
4. 检查 /workspace/ppts/dist/index.html
   ↓
5. 部署：deploy /workspace/ppts/dist
```

## 内容规模规范

### 规模原则

> **4-6 个核心章节即可，章节数不贪多。真正的厚度来自每个章节内部内容的充实度。**

| 页类型 | 数量 | 要求 |
|--------|------|------|
| 封面 | 1 | 简洁，标题+副标题 |
| 目录 | 1 | 三列网格，4-6 个章节 |
| 内容页 | 8-12 页 | **核心，每页 5 个卡片** |
| 结尾 | 1 | 金句收尾 |

**目标：每次生成 16-20 页。**

### 内容充实度要求（强制）

每个卡片的正文必须满足：
- **5-7 句话**（不是 2-3 句）
- 包含**具体案例 / 真实数据 / 场景描述**
- 包含**为什么是这个观点**的逻辑解释
- 避免空洞的结论，要有细节支撑

示例（不合格 vs 合格）：

```
❌ 不好：AI 提升效率，效果显著。（太短，无细节）

✅ 好的：AI 将任务完成时间从「小时级」压缩到「分钟级」，
同等质量下速度提升 5-20 倍。比如一份市场调研报告，人工需要
2-3 天完成，AI 辅助下 2-3 小时即可交付初稿，效率提升 10 倍以上，
这是最直接的效率提升。
```

### 章节规划模板（4-6 个章节，根据主题调整）

```
章节1：概念定义（什么是这个主题，背景是什么，为什么重要）
章节2：核心原理（底层逻辑，为什么它是这样运作的）
章节3：核心能力/优势（详细展开，每点附案例）
章节4：应用场景（典型场景，含完整实施路径和效果数据）
章节5：数据佐证（权威数据，量化说明）
章节6：局限与行动建议（客观分析 + 给观众的具体建议）
```

**注意：不是堆砌章节数量，是把每个核心章节的内容做深做透。**

## 内容页版式标准（默认标准）

> **2026-04-09 确认：以 17 页充实版为准，卡片列表布局为唯一标准版式。**

**标准版式：纵向卡片列表**
- 5 个卡片纵向排列（单列）
- 每个卡片：标题（粗体居中）+ 正文（居中，5-7 句话）
- 白色背景，红色标签徽章，红色分割线
- 这是唯一标准版式，后续所有 PPT 均采用此版式

**禁止使用：**
- ❌ 两栏/三栏分栏布局（屏幕太窄，内容被压缩）
- ❌ 左侧竖线装饰（视觉噪音）
- ❌ 底部独立趋势横栏（与整体风格不一致）

## slide type 一览

| type | 用途 | 关键点 |
|------|------|--------|
| `cover` | 封面 | 只需 title + subtitle，**不要**"封面"标签和翻页提示 |
| `overview` | 目录 | 用 `ov-grid` 三列网格，每项带序号，6-10 个章节 |
| `section` | 内容页 | **5-6 个卡片**，所有文本必须 `text-align:center` |
| `end` | 结尾 | 简洁有力，**白底**，不要多余装饰 |

---

## 封面设计规范

封面**只保留**：
- 主标题（大字，黑）
- 副标题（小字，灰）
- 红色分割线

**禁止出现**：
- "封面" / "Cover" 等标签文字
- 翻页提示语（「← → 键盘翻页」等）

---

## 目录页设计规范

目录使用 **三列网格** 布局，每项有序号（01-10），结构清晰：

```html
<div class="ov-grid">
  <div class="ov-item">
    <div class="ov-num">01</div>
    <div class="ov-text">章节名</div>
  </div>
</div>
```

## 内容页设计规范

### 字号体系

| 元素 | 字号 | 是否居中 |
|------|------|---------|
| 封面标题 cover-title | 3.2-3.4em | yes |
| 页面标题 page-title | 2.0-2.2em | yes |
| 标签徽章 tag-badge | 0.78em | — |
| 卡片标题 card-title | 1.05em | **yes（必须）** |
| 卡片正文 card-body | 0.95em | **yes（必须）** |

> **字号为什么要小？因为内容充实后文字量大，字号小才能放下。宁可小一点，也不要溢出。**

### 布局规则

- **全部白底**：`background:#fff`，禁止深色背景
- **单列卡片**：5-6 个卡片纵向排列，不要横向多列
- **卡片间距**：gap .5em，不要太紧
- **正文行高**：line-height 1.65~1.75，保证可读性

## 铁律（最高优先级，永远遵守）

> ⚠️ **CSS 禁止使用 Python `%` 格式化**
> `width: 100%;` 中的 `%` 与 Python `%` 格式化冲突，会产生 `%%`，导致浏览器 CSS 解析失败，页面全白。
> **CSS 只能写纯字符串拼接（`+`），绝对不能用 `%` 格式化。**

---

## 避坑指南

**① CSS `%%` 与 Python `%` 格式化冲突**
→ CSS 用纯字符串，`+` 拼接，不用 `%` 格式化。

**② 首屏空白**
→ 首屏 `<div class="s active"`，不用 JS 加 active。

**③ 内容太少/太薄**
→ 每次目标 20 页，每页 5-6 卡片，每卡片 5-7 句含案例。宁可多写，不要精简。

**④ 封面过度设计**
→ 只留标题 + 副标题 + 分割线，其他全部不要。

**⑤ 目录页无结构**
→ 三列网格 ov-grid + 序号 ov-num。

**⑥ 文字不居中**
→ `.card-title` 和 `.card-body` 显式加 `text-align:center`。

---

## 核心心法

> **Python % 不碰 CSS** — CSS 中绝对不能用 `%` 格式化
> **首屏硬编码 active** — `class="s active"` 直接写在 HTML 里
> **内容充实 20 页** — 每页 5-6 卡片，每卡片 5-7 句含案例
> **封面极简化** — 只留标题+副标题
> **目录用网格** — ov-grid 三列带序号
